# 제작현황 필터 개편 + 가상화 착수 계획서

`src/components/production/ProductionListMain.tsx` (약 3,819줄)의 필터를 **테이블별 독립 필터**로 개편하고, 동시에 **가상화(성능) 리팩토링**을 함께 진행하기 위한 계획.

> 성능 배경/원인은 [`PRODUCTION_LIST_REFACTORING.md`](./PRODUCTION_LIST_REFACTORING.md) 참고. 이 문서는 그 위에 "필터 개편"을 얹은 실행 계획이다.

## 배경 / 문제

- 현재 필터는 **두 테이블(PCB/Cable)을 하나의 공유 툴바**로 제어(`selectedYear`/`selectedMonth`/`selectedCategories`). PCB와 Cable은 날짜 컬럼·제작구분이 서로 달라 한 필터로 묶기에 부적합.
- 연도 목록이 **하드코딩**(`올해-2 ~ 올해+2`, 397행)이라 데이터와 무관.
- 로딩이 `request_date` 날짜범위(연도) 기준이라 **"전체"가 곧 그 해 전체(2025년 1,063행)를 통째로 렌더** → 렉. 가상화·메모이제이션 부재(memo/useMemo/useCallback 0개).

## 확정된 사양 (사용자 합의 완료)

### 필터를 테이블별로 2개 분리
- **위(PCB) 테이블**: 자체 필터 툴바 — 지금 "요청월" 필터가 있는 그 위치·디자인 그대로 재사용 (이질감 없이).
- **아래(Cable) 테이블**: 위와 **동일한 형태**의 필터 툴바를 Cable 테이블 바로 위에 추가.

### 각 필터 구성 (동일 구조, 대상만 테이블별)
1. **년/월 선택**
   - 날짜 = **년(年)+월(月) 선택만** (자유 입력 없음).
   - **년도 드롭다운은 데이터가 있는 년도만** 표시(현재: 2024·2025·2026). 하드코딩 배열 제거.
   - **기본 = "전체"(년도 필터 미지정 → 전 년도 다 보기).** 사용자가 년도를 고르면 그때 적용. (※ 이전 "현재년도 기본"에서 변경됨)
   - 월은 기존처럼 `전체` + `1~12월` 버튼(badge-stats 스타일) 유지.
2. **기준 날짜 컬럼 드롭다운**
   - 년/월 필터가 "어느 날짜 컬럼"에 적용되는지 지정.
   - **기본값 = 요청일(request_date)**.
   - 드롭다운 목록 = **그 테이블의 날짜형 컬럼만** (아래 표).
   - 혼합컬럼(납품기한·환화·에버텍)도 포함하되, **날짜가 든 행만** 필터에 걸리고 메모 행은 제외.
3. **제작구분 필터** (테이블별 항목 고정)
   - 위(PCB): **LG_PCB, LG_Socket Board, PCB**
   - 아래(Cable): **LG_Cable, LG_Case, Cable, Case** (Cable/Case는 현재 데이터 0건이나 선택지 유지)
4. **입고 상태 필터 (기본 ON, 두 테이블 대칭)** — UI에 보이는 토글
   - 위(PCB): **완제품입고(final_product_stock) = "입고대기"(값 없음)인 항목만** 기본 표시. 토글 끄면 전체.
   - 아래(Cable): **실제입고일(cable_actual_date)이 없는(=입고대기) 항목만** 기본 표시. 토글 끄면 전체.
   - 즉 두 테이블 모두 기본 화면 = "아직 입고 안 된(입고대기) 항목만".

### 입고 컬럼 표시 형식 통일 (완제품입고 + 실제입고일)
**정식 표시 형식 = `MM월 DD일`** ("입고" 텍스트 제거, 볼드 제거, 연도 없음).

현황(완제품입고 final_product_stock, PCB 978건)은 경로별로 형식이 섞여 있음:
| 형식 | 건수 | 처리 |
|---|---|---|
| `MM월 DD일 입고` (버튼 스탬프) | 452 | "입고" 떼고 `MM월 DD일` |
| `2026-05-21` (ISO, 엑셀 임포트) | 344 | `MM월 DD일`로 변환 |
| 빈값 | 124 | 입고대기 버튼 |
| `MM월 DD일 [완료/납품]` (수동) | 53 | 접미사 의미 있음 → 검토(일괄 변환 X) |
| 자유 텍스트/메모 (분할입고 등) | 5 | 그대로 |

- **표시 정규화 방식(권장)**: DB 원본은 두고 **렌더 시 `MM월 DD일`로 표시**하는 포맷 함수 신설.
  - ISO(`YYYY-MM-DD`) → `MM월 DD일`
  - 뒤의 "입고" 단어 제거
  - 순수 날짜가 아니면(완료/납품/메모) 원문 유지
- Cable **실제입고일(cable_actual_date)** 도 위와 **동일 형식·동일 동작**으로:
  - 값 없음 → 파란 **"입고대기" 버튼**, 클릭 시 오늘(KST) 날짜 스탬프.
  - 값 있음 → `MM월 DD일` 표시.
- 신규 입력 형식 통일: `buildStockInLabel`(225행)이 `MM월 DD일 입고` 대신 **`MM월 DD일`** 을 찍도록 변경(앞으로 들어오는 값부터 정식형식).
- 구현: `renderEditableCell`의 `isStockWaiting` 특수처리(1778행)를 `cable_actual_date`까지 확장, `handleStockInPress`가 대상 field를 받도록 일반화(현재 `final_product_stock` 하드코딩, 1164행), 표시 포맷 함수 적용.

### 날짜 컬럼 드롭다운 항목 (코드 실측)
| 위(PCB) | 아래(Cable) |
|---|---|
| **요청일** request_date ★기본 | **요청일** request_date ★기본 |

`*` = "날짜 또는 메모" 혼합컬럼.

## 작업 단계 (순서·의존성)

### Phase 0 — 데이터 로딩 분리 (필터 개편의 전제)
- `loadData`를 **전체 로드**로 전환(연도 하드범위 제거). 날짜필터는 클라이언트 표시 단계로 이동.
  - 이유: 년/월을 "표시 필터"로 쓰려면 데이터가 메모리에 있어야 함. 발주구매관리와 동일한 "전체 로드 + 가상화" 패턴.
  - 사이드 이펙트(긍정): `request_date` NULL 행도 로드됨(현재 숨겨진 3건 등 표시 가능).
- 규모 확인: 전체 ≈ 1,456행 + 증가. 메모리 문제 없음(발주는 2,000건 캐싱 중)이나 **반드시 Phase 2(가상화)와 함께 배포**해야 렉 안 남.

### Phase 1 — 필터 상태·UI 테이블별 분리
- 상태 분리: `selectedYear/Month/Categories` → `pcbFilter`/`cableFilter` 2벌(년·월·제작구분·기준날짜컬럼).
- `years`를 **로드된 데이터 기반**으로 계산(양 테이블의 선택된 기준날짜컬럼에서 연도 distinct, 없으면 현재 연도만). 기본=현재 연도.
- 클라이언트 필터링 함수: `filteredPcbs`/`filteredCables`가 (선택 기준날짜컬럼의 년/월) + (제작구분) 기준으로 거르도록 변경.
- UI: 기존 요청월 툴바(`2013~`)를 PCB용으로 재사용 + Cable 테이블 위(`2938` 직전)에 동일 마크업 복제. 스타일 토큰(badge-stats, 활성색 `#1777CB`) 그대로.
- 기준 날짜 컬럼 드롭다운 신규 추가(위 표대로, 테이블별 그룹).
- **입고 상태 필터 토글 추가(기본 ON)**: PCB=완제품입고 미입고만, Cable=실제입고일 없음만. 클라이언트 필터에 반영.
- **실제입고일(cable_actual_date) 셀을 입고대기 버튼 형식으로 변경**: `isStockWaiting`(1778) 확장 + `handleStockInPress`(1164) field 일반화.

### Phase 2 — 가상화 + 셀 메모이제이션 (성능)
- 행 가상화 도입: `@tanstack/react-virtual`(권장) 또는 사내 `VirtualScrollTable` 재사용 검토.
  - 주의: 제작현황은 **제작구분 그룹 헤더 + sticky 칼럼 + 셀 병합**이 있어 평평한 발주 테이블보다 복잡. 그룹 헤더를 가상 목록의 아이템으로 편입하는 방식 필요.
- 셀/행 컴포넌트 분리 후 `React.memo` — 편집·호버 시 전체 재렌더 방지.
